# Custom Agent Rules

## Commenting Guidelines
Follow the principles from "Clean Code" by Robert C. Martin:
1. **Explain the Why, not the What**: Code should be self-documenting through clear variable and method naming. Comments should explain the intent, business rules, or complex workarounds.
2. **Use XML Documentation**: In C#, always use XML comments (`///`) on public interfaces, classes, and controller endpoints to explain their purpose and usage.
3. **Keep it concise**: Comments should be neither too long nor too short.
4. **Avoid redundant comments**: Do not comment getters/setters or obvious methods (e.g., `// This gets the user` above `GetUser()`).

## Frontend UI/UX Guidelines
1. **Always Use Impeccable Standards**: Whenever building, designing, refactoring, or touching ANY frontend interface (React, CSS, HTML), you MUST apply the principles outlined in the `impeccable` skill.
2. **Execute the Full Lifecycle**: Do not stop at a "minimum viable product" design. You must continuously apply the `impeccable` commands across all phases:
   - **Build**: Use `craft` and `shape` to plan and build features end-to-end. Use `extract` to pull reusable tokens. Use `teach` and `document` to set up context.
   - **Evaluate**: Use `critique` (UX design review) and `audit` (technical/a11y checks) to assess your work.
   - **Refine**: Use `polish` (final quality), `harden` (errors/i18n), and `onboard` (empty states/first-run). For styling adjustments, use `bolder`, `quieter`, or `distill` to tune the aesthetic density.
   - **Enhance**: Elevate the design using `animate` (motion), `colorize` (strategic color), `typeset` (typography), `layout` (spacing/hierarchy), `delight` (personality), and `overdrive` (pushing limits).
   - **Fix & Iterate**: Use `clarify` (UX copy), `adapt` (responsive), `optimize` (performance), and `live` (visual variants) to continuously improve the interface.
3. **Commit to Quality**: Ensure that visual hierarchy, typography, colors, animations, and layouts are flawlessly executed per `impeccable` design laws. Avoid generic "safe" designs and "AI slop" aesthetics (like raw glassmorphism and excessive gradients) unless explicitly requested.
